# Versioning

Format: `Major.Minor.Build`

## Major
Architectural or major-layout release, for example `4.0.0`.

## Minor
Meaningful feature set or algorithm capability, for example `3.54.0`.

## Build
Bug fix, refinement, documentation update, or repackage of the same minor release, for example `3.53.2`.

When Minor increases, Build resets to `0`. When Major increases, Minor and Build reset to `0`.
