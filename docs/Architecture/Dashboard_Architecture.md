# Dashboard Architecture

The dashboard is designed for rapid decision support, not raw data display.

## Information hierarchy
1. Portfolio and market health.
2. Immediate decisions.
3. Selected-stock details.
4. Buy and sell ladders.
5. Opportunity, quality, and trend context.
6. Capital allocation and benchmark evidence.

## Layout constraint
New information should first be integrated by compressing or reorganizing existing cards. New rows are added only when the information cannot remain legible in the existing structure.
