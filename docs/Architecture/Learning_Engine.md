# Learning Engine

## Purpose
Learn from repeated user overrides without blindly copying isolated decisions.

## Required event fields
- system recommendation
- user action
- override reason
- market context
- position context
- later outcome

## Learning stages
1. Record observation.
2. Detect repeated pattern.
3. Estimate outcome difference.
4. Propose strategy-profile change.
5. Require user approval.
6. Monitor the approved rule.
7. Allow rollback.

## Safety rule
The Learning Engine never changes trading rules automatically.
